# BIM-автоматизация: Журнал бетонных работ

> Автоматический конвейер данных: **Revit → Signal API → n8n → Google Sheets**
> 
> Проект сокращает время заполнения исполнительной документации с часов до минут.

---

## 📋 Описание проекта

Конструкторская документация в строительстве требует ведения журналов бетонных работ по форме КС-6а. Ручное заполнение занимает часы и приводит к ошибкам.

**Особенность процесса**: элементы отмечаются как забетонированные в **Navisworks** (в ходе контроля выполнения работ), а не в Revit. Данные из Navisworks синхронизируются с Signal.

**Решение**: автоматический конвейер, который:
1. Получает данные из модели Revit + отметки о бетонировании из Navisworks (через платформу Signal)
2. Преобразует CSV-файл в структурированные данные
3. Сортирует записи по дате укладки бетона
4. Заполняет два Google-таблицы: «Журнал бетонных работ» и «Журнал ухода за бетоном»
5. Отправляет уведомление в Telegram

### Поток данных
1. **Revit** → модель с параметрами элементов (объёмы, марки, отметки)
2. **Navisworks** → отметка элементов как забетонированных (статус выполнения работ)
3. **Signal** → синхронизация данных из Revit + Navisworks, хранение и выдача в формате CSV
3. **n8n** → оркестрация: авторизация, загрузка, парсинг, сортировка, запись
4. **Google Sheets** → два журнала с appendOrUpdate (по ключу «Наименование конструкции»)
5. **Telegram** → уведомление об успешном обновлении
6. **Python** → дополнительный экспорт Google Sheets → Excel

## 🔧 Узлы n8n (подробно)

### 1. Telegram Trigger / Webhook / Manual Trigger
Триггеры запуска:
- **Telegram Bot** — команда `/update_journal`
- **Webhook** — внешний вызов (CI/CD, планировщик)
- **Manual** — ручной запуск из интерфейса n8n

### 2. DOCS_CSV (Set node)
Параметры документа:
```json
{
  "project_name": "ЖК МЫ поз.6.2",
  "folder_name": "6. СМР/Журнал бетонных работ",
  "file_name": "12.csv"
}
```

### 3. DownloadFileFromDocs (Code node)
JavaScript-код для работы с Signal API:
- Авторизация по `clientId` + `clientSecret` + scopes
- Поиск проекта по имени
- Рекурсивный обход папок (`folder1/folder2/folder3`)
- Получение `itemId` → `versionId` → `signedUrl`
- Скачивание файла

**Ключевые функции:**
```javascript
// Нормализация имён (регистронезависимый поиск)
const sameName = (a, b) => normalize(a).toLowerCase() === normalize(b).toLowerCase();

// Рекурсивный проход по пути папок
const getFolderByPath = async (jwt, startFolderId, path) => { ... };

// Два варианта задания пути:
// 1) folder_name = "Папка1/Папка2", file_name = "Файл.csv"
// 2) folder_name = "", file_name = "Папка1/Папка2/Файл.csv"
```

### 4. HTTP Request → Extract from File
- Загрузка CSV по `signedUrl` (формат ответа: file)
- Извлечение данных: разделитель `;`, кодировка UTF-8 с BOM

### 5. Parse & Sort CSV (Code node)
```javascript
// Очистка ключей от переносов строк
const cleanKey = key.replace(/[\r\n]/g, ' ').trim();

// Сортировка по дате в формате ДД.ММ.ГГГГ
const parseDate = str => {
  const [day, month, year] = str.split('.');
  return new Date(year, month - 1, day);
};
```

### 6. Журнал бетонных работ (Google Sheets node)
Операция: `appendOrUpdate`
- **Ключ поиска**: `Наименование бетонируемой конструкции и ее расположение`
- **Поля** (маппинг из CSV):
  - Дата и время укладки → `SIGNAL Completed`
  - Наименование конструкции → `Объект Имя + Маркер + Отметки + Комментарии + ADSK_Этаж`
  - Изготовитель → `SIGNAL Contractor`
  - Номер партии → `SIGNAL Номер партии`
  - Объём → `Объект Объем`
  - Температура → `SIGNAL Температура`
  - Способ твердения → константа «Естеств. условия»
  - Класс прочности → `Тип в приложении Revit Описание`
  - Процент прочности → `SIGNAL процент`
  - Подпись → константа «Чепкасов С. А.»

### 7. Журнал ухода за бетоном (Google Sheets node)
Аналогично, но другой набор полей:
- Наименование части сооружения
- Дата укладки
- Объём бетона
- Метод выдерживания
- Температуры
- Дата и время начала выдерживания
- Продолжительность (константа «3 сут.»)
- % теоретической прочности
- Ф.И.О. ответственного

### 8. Уведомление в Telegram

## 🚀 Как использовать

### Требования
- n8n (self-hosted или cloud)
- Аккаунт Signal (sgnl.pro) с API-доступом
- Google Sheets API credentials (OAuth2)
- Telegram Bot (BotFather)

### Установка
1. Импортируйте workflow JSON в n8n
2. Настройте credentials:
   - Signal API (clientId, clientSecret)
   - Google Sheets OAuth2
   - Telegram Bot API
3. Укажите `documentId` и `sheetName` для ваших таблиц
4. Настройте параметры проекта в узле `DOCS_CSV`

### Запуск
- Через Telegram: отправьте `/update_journal`
- Через Webhook: `POST` на URL webhook
- Вручную: кнопка «Execute workflow» в n8n

---

## 🔐 Безопасность

- Все credentials хранятся в зашифрованном виде в n8n
- `clientSecret` и токены не попадают в логи
- Доступ к Google Sheets через OAuth2 с ограниченными scopes
- Для production рекомендуется использовать переменные окружения
